# Справка: статьи затрат и категории

Снято с боевой базы 22.07.2026. Нужно для модуля 6 (дашборды, раздел 7 ТЗ)
и модуля 7 (карточка поставщика).

Это **две независимые оси аналитики** на `purchase.order`:

- `cost_analytic_account_id` — **Статья затрат**, «на что потратили»
- `cost_category_id` — **Категория**, «подо что потратили» (направление продукции)

Обе обязательны при подтверждении закупки (`button_confirm` их проверяет).

**Заполненность:** статья — у 32 из 32 заказов, категория — у 29 из 32.

### Ловушка при группировке

Планы `1.10 Материал`, `1.11 Аутсорс (прочие)`, `1.12 Комплектующие`
существуют **одновременно** как подпланы (id 11, 12, 13) и как статьи
с теми же кодами внутри плана «1. Производственные».

Для Odoo это нормально, но при группировке по `cost_plan_id` картинка
задвоится. **Группировать по `cost_plan_root_id` и
`cost_analytic_account_id`.** См. п. 7.4 ТЗ.

### Верхний уровень — главный разрез

`Производственные / Общефирменные / Инвестбюджет` — это по сути
«текущая деятельность против инвестиций». Первое, на что смотрит
собственник. Считается подъёмом по `account_analytic_plan.parent_id`
до корня (`cost_plan_root_id`, рекурсивный CTE в SQL-view).

## Как получить актуальный срез

```sql
-- планы
SELECT id, name, parent_id FROM account_analytic_plan ORDER BY id;

-- счета с планами
SELECT a.id, a.name, a.code, p.name AS план
FROM account_analytic_account a
LEFT JOIN account_analytic_plan p ON p.id = a.plan_id
ORDER BY p.id, a.code;

-- распределение по статьям
SELECT a.name, count(*), round(sum(o.amount_total)::numeric, 2)
FROM purchase_order o
LEFT JOIN account_analytic_account a ON a.id = o.cost_analytic_account_id
GROUP BY a.name ORDER BY 3 DESC NULLS LAST;
```
