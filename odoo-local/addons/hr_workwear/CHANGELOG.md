# Changelog

Все заметные изменения модуля «Спецодежда».

## 2026-07-24

- Первая версия: установлен штатный `hr`. Справочник Б (`hr.department`,
  14 подразделений из реестра личного состава) + сопоставление со
  справочником А (`purchase.request.department.hr_department_id`, ручное,
  заполняется заказчиком). `post_init_hook` вносит согласованные с
  заказчиком (22.07.2026) правки: переименование «Технологический отдел» →
  «Инженерный отдел», перенос Наточисела на «Отдел управления».
- 13 `hr.employee` созданы автоматически из активных `res.users` (данные
  из реестра, сопоставление по логину); ещё 53 человека - в
  `data/hr_employee_import.csv` для штатного импорта заказчиком. Федчин
  (собственник) и технический логин `admin` - `workwear_not_required`.
- Размеры (`hr.workwear.type`/`.size`/`hr.employee.workwear.size`, 8
  стартовых типов СИЗ), выдача (`hr.workwear.issue`/`.line` -
  автоподстановка размера, списание со склада кнопкой), нормы и
  ежедневное напоминание кладовщику о просрочке (`ir.cron` +
  `activity_schedule`).
- Приложение «Спецодежда»: Личный состав, Выдачи, Просрочено, Потребность
  (+ создание заявки на закупку), Расход. Два QWeb-отчёта (личная карточка
  учёта СИЗ, ведомость выдачи).
- Права: `group_workwear_manager`, обычный сотрудник видит только свою
  карточку/выдачи (`ir.rule`, ПДн).
- Попутно исправлен реальный конфликт имён полей: `purchase_pdf_import`
  и штатный `hr` независимо заводили `res.users.department_id` с разным
  смыслом - Odoo молча склеивала оба определения в одно битое поле.
  Переименовано в `purchase_pdf_import` → `purchase_department_id`,
  старые 5 значений перенесены миграцией в `post_init_hook` этого модуля.
- 22 теста (12 методов), 0 failed, 0 error. Полная регрессия по всем 9
  модулям - 195 тестов, 0 failed, 0 error. Чистый uninstall/reinstall
  проверен.
- Загружены оставшиеся 53 человека из реестра (напрямую, минуя мастер
  импорта Odoo) - `hr.employee` теперь 66 записей и локально, и на
  сервере.
- Добавлена классификация `contact_kind` на `res.partner` (Сотрудник/
  Поставщик/Клиент/Прочее) + фильтры и группировка «Тип контакта» поверх
  штатного поиска «Контакты» - по просьбе пользователя, чтобы 66 рабочих
  контактов сотрудников не смешивались визуально с поставщиками. Сам
  механизм создания рабочих контактов (штатный `hr`) не тронут - см.
  NOTES.md. 3 новых теста, 27 тестов модуля всего - 0 failed, 0 error.
  Полная регрессия - 198 тестов по всем 9 модулям, 0 failed, 0 error.
  Uninstall/reinstall проверен ещё раз (штатная вьюха "Контакты"
  возвращается к исходному виду после удаления модуля).
