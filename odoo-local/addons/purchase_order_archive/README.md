# purchase_order_archive

Модуль из раздела 4 ТЗ, четвёртый по очереди внедрения (п. 11.1) — вводит
`is_completed`, нужный модулю `purchase_registry_ux` (пятому по очереди).

## Что делает

- **`is_completed`** (stored compute) — закупка получена (`in_stock`),
  оплачена на 95%+, документы в порядке (`document_status == 'done'`),
  не отменена и не отклонена. `completed_date` фиксируется один раз при
  первом переходе в `True` и больше не меняется, даже если условие потом
  временно перестанет выполняться.
- **`is_archived`/`archived_date`** — свой флаг, не родное поле Odoo
  `active` (у `purchase.order` в Odoo 18 такого поля нет, и добавлять не
  нужно — это включило бы ORM-фильтрацию глобально, архивные закупки
  пропали бы из стандартного приложения «Покупки», отчётов, связанных
  `stock.picking` и `request_ids` заявок).
- Реестр закупок по умолчанию скрывает архивные (фильтр «Скрыть архивные»,
  включён по умолчанию) — тумблер «Показать архивные» рядом снимает
  ограничение, не уходя в отдельное меню.
- Новое меню **«Архив закупок»** — та же вьюха реестра, домен
  `is_archived = True`.
- `ir.cron` раз в сутки архивирует завершённые закупки через 5 **рабочих**
  дней после `completed_date` (по `resource.calendar` компании; без
  календаря — фолбэк пн-пт без учёта праздников, с предупреждением в лог).
- Кнопки «В архив» / «Вернуть из архива» на форме заказа
  (`group_chief_buyer` + `purchase.group_purchase_manager`).
- Фолбэк-колонка «Завершена» (`is_completed`, `widget="boolean_toggle"`,
  readonly, `optional="hide"`) в реестре/архиве — см. `NOTES.md` про то,
  что произойдёт, когда появится `purchase_registry_ux`.

Файлы `purchase_pdf_import` не тронуты.

## Как проверить руками

1. Установить модуль.
2. Довести тестовую закупку до получения + полной оплаты + УПД (или
   пропуска УПД) — `is_completed` должно стать `True` (галочка в фолбэк-
   колонке реестра).
3. Нажать «В архив» на форме заказа — закупка пропадает из «Реестра
   закупок», но видна в «Архиве закупок» и по прямой ссылке.
4. В «Реестре закупок» включить «Показать архивные» — архивная закупка
   должна снова появиться в общем списке.
5. «Вернуть из архива» — закупка снова в обычном реестре.
6. Вручную запустить cron «Архивация завершённых закупок» (Настройки >
   Технические > Автоматизация > Запланированные действия) на закупке с
   `completed_date` больше 5 рабочих дней назад — должна архивироваться
   автоматически с сообщением в чате.

## Тесты

`tests/test_order_archive.py` — неполная оплата не даёт `is_completed`,
`completed_date` не перезаписывается при повторном пересчёте, фолбэк-функция
рабочих дней корректно пропускает выходные, cron не архивирует раньше срока
и архивирует после, архивная закупка не видна в реестре/видна в архиве/
доступна по id, архивация не рвёт `request_ids` и связь со `stock.picking`.
