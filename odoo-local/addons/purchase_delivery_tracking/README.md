# purchase_delivery_tracking

Модуль из раздела 5 ТЗ, третий по очереди внедрения (п. 11.1) — небольшой,
изолированный.

## Что делает

- Справочник **`purchase.delivery.method`** (способ доставки): Силами
  поставщика, Самовывоз, Яндекс Доставка, СДЭК, Деловые Линии, ПЭК, Почта
  России, Транспортная компания (иное), Курьер. Список стартовый, не
  закрытый — быстрое создание нового способа разрешено прямо из поля.
- На `purchase.order`: `delivery_method_id`, `tracking_number` (свободный
  текст — код или пояснение, необязателен), `shipped_date`,
  `tracking_url` (compute, ссылка для СДЭК/Почты России/Деловых Линий/ПЭК,
  если номер похож на код, а не на текст), `delivery_summary` (compute, для
  колонки реестра).
- Кнопка «Оформить доставку» (на форме заказа и на форме заявки) больше не
  дёргает `action_arrange_delivery` напрямую через `confirm=` — открывает
  новый wizard **`purchase.delivery.tracking.wizard`**: способ обязателен,
  номер — нет, поле номера прячется для способов без трек-кода (но не
  очищается при переключении). «Подтвердить» записывает поля на заказ и
  вызывает существующий `action_arrange_delivery()` как есть.
- Чат заказа и каждой его заявки получает отдельное сообщение с деталями
  доставки («Доставка оформлена — СДЭК, трек-номер 1234567890.» /
  «Доставка оформлена — Силами поставщика.») в дополнение к штатному
  сообщению `action_arrange_delivery`.
- Форма заявки показывает способ/номер/дату отправки **только на чтение**.
- Реестр закупок: колонка «Доставка» (`optional="hide"`) и группировка по
  способу доставки в поиске.

Файлы `purchase_pdf_import` не тронуты — только `_inherit` моделей и
XML-наследование вьюх.

## Как проверить руками

1. Установить модуль.
2. На согласованном и оплаченном заказе (кнопка «Оформить доставку»
   видна) нажать её — должен открыться wizard, а не диалог подтверждения.
3. Выбрать способ без трек-кода (Самовывоз) — поле номера должно исчезнуть.
   Переключить на СДЭК — поле должно появиться, а если что-то уже было
   введено раньше при другом способе, значение не должно потеряться.
4. Подтвердить с указанным номером — в чате заказа и связанной заявки
   должно появиться сообщение с названием способа и номером.
5. Открыть заявку — поля доставки видны, но не редактируются.
6. В «Реестре закупок» включить скрытую колонку «Доставка» через меню
   колонок и сгруппировать по способу доставки.

## Тесты

`tests/test_delivery_tracking.py` — оформление без номера для способов без
трек-кода, сохранение номера при переключении способа, разное содержимое
сообщения в чате в зависимости от наличия номера, быстрое создание нового
способа не создаёт дублей при повторном вводе, `tracking_url`/
`delivery_summary` считаются корректно.
